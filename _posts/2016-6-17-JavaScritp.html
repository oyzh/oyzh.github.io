JavaScript和CSS一样修改静态的HTML，以使得网页有动态的效果。DHTML(动态HTML)只适用于Web文档，“DOM脚本程序设计”则涵盖了任何一种支持DOM API的程序设计语言去处理任何一种标记文档的所有情况。具体到Web文档，JavaScript语言的特点使它成为了DOM脚本程序设计的最佳选择。

简单语法

  • 字符串可以和数字相加，此时数字直接转换为字符串。
  • 函数 
function shout(){
  var beatles = Array("John, "Paul", "George", "Ringo")
  for (var count = 0 ; count < beatles.length; count++)
    alert(beatles[count]);
  }
}
  • 作用域：如果在某个函数中使用了var，那个变量就将被视为一个局部变量，它将只存在于这个函数的上下文中;反之，如果没有使用var，那个变量就将被视为一个全局变量，如果脚本里已经存在一个与之同名的变量，这个函数将覆盖那个现有变量的值。
  • 对象：内建对象，宿主对象。由Web浏览器提供的预定义对象被称为宿主对象。宿主对象包括Form、Image和Element。我们可以通过这些对象获得关于某给定网页上的表单、图像和各种表单元素的信息。

DOM

当创建一个网页并把它加载到Web浏览器中时，DOM就在幕后悄然而生。它将根据编写的网页文档创建一个文档对象。DOM代表着被加载到浏览器窗口里的当前网页：浏览器向我们提供了当前网页的地图（或者说模型），而我们可以通过JavaScript去读取这张地图。

DOM把一份文档表示成一棵树，如果把各种文档元素想象成一颗家谱树上的各个节点的话，我们可以用同样的记号来描述DOM。

document.getElementById()
document.getElementByTagname()
getAttribute()
setAttribute()

DOM的工作模式：先加载文档的静态内容，再以动态方式对它们进行刷新，动态刷新不影响文档的静态内容。这正是DOM的真正威力和诱人之处：对页面内容的刷新不需要最终用户在他们的浏览器里执行刷新操作就可以完成。

CSS

DOM并不是人们与网页结构打交道的唯一手段。我们还可以通过CSS告诉浏览器应该如何显示一份文档的内容。类似JavaScript脚本，对样式的声明即可以嵌在文档的<head>部分（这需要用到<style>标签），也可以放在另外一个样式表文件里。

在某些场合，当把样式应用于一份文档时，我们其实只想让那些样式只作用于某个特定的元素。为了获得惊喜的控制，我们将需要在文档里插入一些能够把这段文本与其他段落区别开来的特殊标记。

为了把某一个或某几个元素与其他元素区别开来，我们需要使用class属性或id属性之一。 

<p class="special">this paragraph has the special class</p>
<h2 class="special">So does this headline</h2>

在样式表里，我们可以像下面这样为class属性值相同的所有元素定义一种共享的样式： 

.special {
  font-sytle: italic;
}

id属性的用途是给网页里某个元素加上一个独一无二的标识符： 

<ul id="purchases">

在样式表里，我们可以像下面这样为有着特定id属性值的元素定义一种独享的样式： 

#purchases{
  border: 1px solid white;
  background-color: #333;
  color: #ccc;
  padding: 1em;
}
